# 도커란?
도커 설치와 관련된 포스팅 이기에 아주 간단히 도커가 어떤 것이지  살펴만 보고 바로 설치 하는 방법을 알아보자.


 - 도커 공식 홈페이지에는 "Docker는 컨테이너(Container) 이동을 주도하는 회사, 하이브리드 클라우드의 모든 애플리케이션을 처리할 수 있는 유일한 컨테이너 플랫폼 제공 업체" 라고 되어 있다.

 

 - 도커는 컨테이너 기반의 오픈소스 가상화 플랫폼 이다.

 

 - 컨테이너란 "다양한 OS에 여러 application이 올려져 있는 것 이라고 볼 수 있다.

 - 도커에서 의미하는 컨테이너란

    택배나 화물선 위에 수출 용품을 싣는 컨테이너를 대신하여 프로그램(소프트웨어)을 담는 격리된 공간을 의미 한다.

 - 각각의 격리된 여러개의 소프트웨어 컨테이너에는 ubuntu, centos등의 운영체제,

   그리고  java, python, 아파치 웹서버나, mysql 같은 dbms 등의 프로그램들 까지도 담기게 된다.

 - 그리고 각 컨테이너는 격리된 공간이기 때문에 한 컨테이너가 해킹을 당하던, 한 컨테이너에 문제가 생기더라도 컨테이너간에 영향을 끼치지 않는다.

 

이러한 도커의 특성 때문에 다음과 같은 장점이 있다고 정리할 수 있다.


# 특징
## 빠르고 가벼운 가상 솔루션.
 - 호스트의 운영체제를 공유하여 필요한 최소한의 리소스만 할당받아 동작하는 방식 이기때문에 가볍다. 
 - 즉 ,기존 Hypervisor 엔진을 사용하지 않고, Docker Engine을 통해 Guest OS 없이 실행 가능하다. Guest OS가 없기 때문에 가상머신보다 훨씬 빠른 실행 속도를 보장할 수 있다. 

 - 하드웨어 가상화를 하지 않기 때문에, 메모리 엑세스, 파일 시스템, 네트워크 실행 성능도 뛰어나다.

## 개발언어에 종속되지 않는다.
- 개발 언어나 툴에 상관없이 어떠한 애플리케이션이라도 만들 수 있으며, Docker에서 만들어진 애플리케이션은 이동성이 뛰어나며 어디서나 실행될 수 있다.

## 뛰어난 보안성.
 - 서비스가 해킹시에도 각 컨테이너들은 격리되어있기 때문에 원래의 서버에 영향을 미치지 않는다.

# 실행
Docker 를 실행하면 아래의 명령어를 쳐보라고 나오는데, CMD 를 열어서 아래 명령어를 쳐보자
```
docker run -d -p 80:80 docker/getting-started
```
[![Docker 시작하기](/assets/imgs/docker/7.png)](/assets/imgs/docker/7.png)

도커 화면
[![Docker 시작하기-화면](/assets/imgs/docker/8.png)](/assets/imgs/docker/8.png)


여기서 화면에 보이는 탭중에 ```Images``` 를 클릭하면 내가 만든 도커 컨테이너가 보이고. 마우스를 올리면, 우측에 ```...```과 ```run``` 이 보인다.
[![Docker 시작하기-화면](/assets/imgs/docker/9.png)](/assets/imgs/docker/9.png)

... 에는 ```pull```, ```push to hub``` 등이 보이는데, pull 은 컨테이너를 관련된 도커라이브러리를 최신화 하는것이고, push to hub 는 말 그대로 container 의 스냅샷을 dockerhub 로 올려 저장하는 것이다.

```in use```를 누르면, 내가 만든 컨테이너들이 보이게 된다.


# Nginx 설치
CMD 창을 열어 아래 명령어를 실행하자
 ```
docker pull nginx
```
[![Docker 컨테이너 클릭](/assets/imgs/docker/11.png)](/assets/imgs/docker/11.png)

처음 에러는 도커를 실행하지 않고 명령어를 실행했을 때나는 오류이다. 도커를 실행하고 명령어를 실행하자.

설치확인
```
docker image ls
```

# Nginx 실행
```
docker container run --name webserver -d -p 80:80 nginx
```
명령어를 실행하고 나면 도커화면에서 RUN 표시가 뜨게 된다.
[![Docker Nginx 실행](/assets/imgs/docker/12.png)](/assets/imgs/docker/12.png)
또 명령어 바로 아래에 보이는 엄청 긴 영문+숫자가  ```컨테이너ID``` 이다.

브라우저에서 ```localhost:80```을 쳐보자
[![Docker Nginx 실행](/assets/imgs/docker/13.png)](/assets/imgs/docker/13.png)

- Nginx 상태 확인
```
docker container ps
```
- 컨테이너 가동 확인
```
docker container stats webserver
```
- Nginx 중지
```
docker stop webserver
```
- Nginx 가동
```
docker start webserver
```
