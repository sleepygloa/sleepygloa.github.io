블로그의 기본 구성은 최상위 폴더에 있는 ```_config.yml``` 을 가지고 설정한다.

기본적으로 많이 사용하는 기능들 위주로 작성하였습니다.

# 1. 기본 구성
약 283 줄의 설정들을 볼 수가 있는데, 제일처음 볼수 있는 것들은 아래와 같다.
```
# Site Settings
locale                   : "en-US" 
title                    : "Site Title"
title_separator          : "-"
subtitle                 : # site tagline that appears below site title in masthead
name                     : "화면에 보여줄 저의 네임(닉네임)."
description              : "네임 밑의 저를 표현할 설명."
url                      : "https://mmistakes.github.io"
baseurl                  : # 서브 경로가 있는 경우 기재 "/blog"
```
사이트의 이름, 나의이름, URL 등 사이트를 나타낼 간단한 구성이 제일 위에 표현된다.
```#``` 으로 주석처리된 부분은 참고하고, 수정하지않을 경우 그냥 ```#``` 상태로 두어도 잘 작동된다.


# 2. teaser와 logo 그림 파일
```
teaser                   : "/assets/images/picture.jpg"
logo                     : # 최상단 메뉴 바에 사이트 로고 넣기
```
```teaser``` 는 왼쪽메뉴의 닉네임 위에 표현할 동그라미 이미지 이다.
```logo``` 는 최상단 메뉴 바에 사이트로고를 나타낸다.

```teaser``` 에서 보다시피 ```/asserts/``` 경로에서 동적 파일을 관리한다. (js, css, font, images)...

각 페이지에서 ```url``` 을 쉽게 불러와 사용하고 싶다면 아래와 같이 사용하면 된다.
```
![]({{ site.url }}{{ site.baseurl }}/assets/images/picture.jpg    )
```

# 3. 블로그 댓글 기능
```
comments:
  provider               : # 블로그 댓글 기능 false (default), "disqus", "discourse", "facebook", "staticman", "staticman_v2", "utterances", "custom"
  disqus:
    shortname            : # 블로그 댓글 기능 https://help.disqus.com/customer/portal/articles/466208-what-s-a-shortname-
  discourse:
    server               : # https://meta.discourse.org/t/embedding-discourse-comments-via-javascript/31963 , e.g.: meta.discourse.org
  facebook:
    # https://developers.facebook.com/docs/plugins/comments
    appid                :
    num_posts            : # 5 (default)
    colorscheme          : # "light" (default), "dark"
  utterances:
    theme                : # "github-light" (default), "github-dark"
    issue_term           : # "pathname" (default)
```

블로그 댓글 기능 disqus, discourse, facebook, staticman, utterances 정도가 대표적이다. GitHub Pages 자체적으로 댓글을 제공하고 있지 않기때문에 보통 외부 댓글 서비스를 연결해주는 방식을 사용한다. 

# 4. 오픈그래프 이미지 등록
SNS 등에서 어떤 링크에 대한 미리보기나, 제목, 설명등을 구성할때 표현하는 ```메타 데이터 표기 방법``` 이다.
누군가에게 내 블로그 글을 SNS 을 통해 공유하게 된다면,  카카오톡에서 사이트 링크걸때 표현되는 이미지 같은 느낌으로 보여 줄 수 있다.
```
og_image                 : "/assets/images/picture.jpg"
```

# 5. 구글 어날리틱스 등록
jekyll 블로그에서는 구글 애널리틱스 를 연결하여 사용 할 수 있다.
```
# Analytics
analytics:
  provider               : false # 구글 어날리틱스 등록 false (default), "google", "google-universal", "custom"
  google:
    tracking_id          :
    anonymize_ip         : # true, false (default)
```

# 6. 사이트 저자 소개
```
# Site Author
author:
  name             : "꾸준하게 개발HA자"
  avatar           : "/assets/images/picture.jpg"
  bio              : "취미로 코딩하는 개발자"
  location         : "South Korea"
  email            :
  links:
    - label: "Email"
      icon: "fas fa-fw fa-envelope-square"
      url: mailto:useremail@gmail.com
    - label: "Website"
      icon: "fas fa-fw fa-link"
      url: "https://usersite.com"
```
minimal-mistakes theme은 사이트 좌측 사이드바에 기본으로 사이트 저자 소개 항목이 존재한다. 물론 이를 간단한 설정으로 없애버릴 수 있다. 저자 소개를 할 생각이면 여기에 간단한 소개를 적어보자. 여러가지 SNS 정보를 등록할 수도 있다. SNS 정보를 등록하면 SNS Icon으로 해당 정보를 표시해준다.

# 7. 사이트 저자 소개 - Footer
웹 사이트의 맨하단에 표시되는 영역을 말한다.
jekyll 블로그를 만들면 저자정보, SNS 링크 등 모두 Footer 에 달수 있다. GITHUB의 경우는 기본으로 달려있다.
```
# Site Footer
footer:
  links:
    - label: "GitHub"
      icon: "fab fa-fw fa-github"
      url: "https://github.com/username"
```

# 8. 블로그 표시 방법 설정
```
# Outputting
permalink: /:categories/:title/
paginate: 5 # 첫 페이지에보여줄 최근 게시물 수를 지정
paginate_path: /page:num/
timezone: Asia/Seoul # https://en.wikipedia.org/wiki/List_of_tz_database_time_zones
```
```paginate``` 는 첫페이지에서 보여주는 recent posts의 개수이다. 이 개수를 넘어서는 오래된 게시물들은 다름 페이지 번호로 넘어간다.
```timezone``` 은 한국이면 Seoul을 사용한다.

# 9. _posts, _pages 기본 설정
```
# Defaults
defaults:
  # _posts
  - scope:
      path: ""
      type: posts
    values:
      layout: single
      author_profile: true
      read_time: true
      comments: # 블로그 댓글 기능 true
      share: true
      related: true

  # _pages
  - scope:
      path: ""
      type: pages
    values:
      layout: single
      author_profile: true
      read_time: false
      comments: false
      share: true
      related: false
```
jekyll 에서의 게시물은 크게 post와 page 로 구분된다.
post 는 날짜 기반으로 게시가 되고, page는 날짜기반으로 게시하지 않는다.

지킬에서의 게시물은 크게 post와 page로 구분된다.
그 외에는 다른 자세한 내용이 많지만, 그부분만 알면 내가 원하는 블로그를 만드는데는 큰 문제가 없을 것이다.

저는 post 형식의 블로그는 제가 작성하는 기술블로그에 적합하지 않다고 생각하기 때문에
첫페이지부터 page 를 연결하여 큰 카테고리와 그외에 순서대로 글을 작성해 나가려 합니다.
