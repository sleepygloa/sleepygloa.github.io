# 2. page 고르기
## 사용법
```test/_pages``` 폴더 안에서 원하는 페이지를 고른 다음, 최상단에 동일한 이름으로(또는 지정할 이름) 폴더 생성을 한다.
그 이후 ```_data/navigation.yml``` 에 title과 url 을 설정하여 경로 설정을 완료하고,
```test/_pages``` 에서 고른 페이지를 복사하여 새로만든 폴더에 복사하자.

## page 내용
```_data/navigation.yml``` 에 주석 처리 되어있던 ```year-archive.yml``` 을 살펴보자.
```test/_pages/year-archive.md``` 파일을 보면 아래와 같은 ```yml``` 설정이 있다.
```
---
title: "Posts by Year"
permalink: /year-archive/
layout: posts
author_profile: true
---
```
하나씩 살펴보면
- title : 페이지 제목
- permalink : 페이지 제목 옆 링크가 표시될때의 URL 
- layout : 어떤 데이터들을 보여줄 것인지(default:post)
- author_profile :  저자 프로필 표시 여부

이렇게 작성되어있다. 사실 특별한 경우를 제외하고는 수정할 필요없다.

# 3. 404 page 등록하기

```
---
title: "Page Not Found"
excerpt: "Page not found. Your pixels are in another canvas."
permalink: /404.html
author_profile: false
---

요청하신 페이지를 찾을 수 없습니다.

<script>
  var GOOG_FIXURL_LANG = 'en';
  var GOOG_FIXURL_SITE = 'https://devinlife.com'
</script>
<script src="https://linkhelp.clients.google.com/tbproxy/lh/wm/fixurl.js">
</script>
```
위 내용은 예제에서 사용한 404.md 파일이다. GitHub Pages에서 404.html 페이지는 주소를 찾을 수 없을때 기본으로 보여주는 페이지이다.

블로그 내에서 정의되지 않은 주소로 접속을 시도하면 GitHub Pages는 기본 Not Found 페이지를 보여주게 되는데, 블로그에 404.html 주소를 등록해두면 GitHub Pages는 기본 Not Found 페이지가 아닌 블로그에서 등록한 404.html 페이지를 보여준다.

여기서는 일부러 author_profile: false 설정을 해보았다. https://username.github.io/404.html 주소를 접속해보자. 사이드 바의 작가 소개항목이 표시되지 않는 것을 알 수 있다. 그리고 입력한 페이지를 찾을 수 없으니 해당 블로그에서 구글 검색을 할 수 있는 검색창을 보여준다.

이렇게 page 글을 사용하면 블로그의 필요한 구성 페이지를 마음대로 만들 수 있다.



원하는 페이지의 구성이 완료되었다면, 샘플로 두었던 post 는 모두 지우자.
제가 사용할 페이지는 ```about```, ```category-archive``` 입니다.
